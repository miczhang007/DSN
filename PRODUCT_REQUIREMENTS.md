# 桌面便签产品需求文档

## 1. 产品概述

### 1.1 产品名称

- 中文名：桌面便签-本地开源版
- 英文名：StickyNote
- 应用文件名：stickynote

### 1.2 产品定位

桌面便签是一款 Windows 本地桌面待办工具，以便签形态常驻桌面，用于快速记录、查看、完成和回溯个人待办任务。

产品强调轻量、本地化、低打扰和快速操作，不依赖服务端账号体系，不做联网鉴权，通过本地用户管理区分不同用户的数据。

### 1.3 目标用户

- 需要在桌面上长期保留待办提醒的个人用户。
- 需要按不同使用者或场景隔离本地任务数据的用户。
- 偏好简单、无需登录鉴权、无需云同步的单机工具用户。

### 1.4 当前版本

- 版本号：v1.0
- 版本展示格式：`v1.0 - YYYY-MM-DD HH:mm`
- 更新时间规则：每次修改应用代码后，必须同步更新 `src/App.vue` 中的 `versionLabel`，时间取本次代码修改完成时间。
- 源码仓库：https://github.com/miczhang007/DSN

## 2. 产品目标

- 提供一个打开即可使用的桌面待办入口。
- 支持任务新增、编辑、完成、撤销完成和生命周期查看。
- 支持本地多用户数据隔离。
- 支持便签主题、大小、位置和开机自启动设置。
- 所有核心数据保存在本机，避免引入服务端依赖。

## 3. 平台与技术边界

- 运行平台：Windows 桌面端。
- 应用形态：Tauri 桌面应用。
- 前端技术：Vue + Vite。
- 本地存储：SQLite。
- 安装包形态：NSIS 安装包。
- 输出文件：
  - `outputs/stickynote.exe`
  - `outputs/stickynote-setup.exe`

## 4. 核心用户流程

### 4.1 新增待办

1. 用户在首页点击添加按钮。
2. 进入新增任务页。
3. 输入任务内容。
4. 可选维护截止时间。
5. 可选标记紧急。
6. 点击添加任务。
7. 返回首页并展示新增任务。

### 4.2 完成待办

1. 用户在首页点击任务左侧完成按钮。
2. 任务从待办列表移除。
3. 任务进入历史任务列表。
4. 系统记录完成和归档生命周期事件。

### 4.3 查看和编辑待办

1. 用户点击首页待办任务内容。
2. 进入任务详情页。
3. 可编辑任务内容、截止时间和紧急状态。
4. 点击保存变更。
5. 系统记录变更生命周期事件。

### 4.4 查看历史任务与撤销完成

1. 用户从菜单进入历史任务。
2. 页面展示已完成任务列表。
3. 用户可点击任务查看生命周期。
4. 用户可点击撤销完成。
5. 撤销后任务重新回到待办列表。

### 4.5 用户管理

1. 用户从菜单进入用户管理。
2. 默认收起新增用户输入栏。
3. 点击添加用户后展开输入栏。
4. 支持新增用户、切换用户、修改用户名、删除用户。
5. 删除用户前必须有明确提示，提醒删除后不能恢复用户数据。

## 5. 功能需求

### 5.1 首页待办列表

- 展示当前用户的未归档任务。
- 任务排序规则：
  - 紧急任务优先。
  - 有截止时间的任务优先于无截止时间任务。
  - 截止时间更早的任务优先。
  - 创建时间更早的任务优先。
- 任务未维护截止时间时，不展示截止时间信息。
- 任务标记紧急时，展示紧急标识。
- 空列表时展示空状态，并提供添加入口。
- 首页标题为 `Today`，标题样式应弱化，不抢占主体内容。

### 5.2 新增任务

- 必填：任务内容。
- 可选：截止时间。
- 可选：紧急标记。
- 任务内容最大长度：80 字符。
- 任务内容为空时，添加按钮不可用。

### 5.3 任务详情

- 待办任务详情支持编辑：
  - 任务内容。
  - 截止时间。
  - 紧急状态。
- 已归档任务详情不允许编辑，仅展示归档摘要和生命周期。
- 任务详情页展示生命周期列表。
- 生命周期应记录：
  - 创建任务。
  - 任务内容变更。
  - 截止时间变更。
  - 紧急标记变更。
  - 完成任务。
  - 归档任务。
  - 撤销完成。

### 5.4 历史任务

- 展示当前用户已归档任务。
- 按归档时间倒序展示。
- 历史页标题为 `History`，标题样式应弱化，不抢占主体内容。
- 每条历史任务展示任务内容和完成时间。
- 支持撤销完成。
- 撤销完成后：
  - 清空任务的完成时间和归档时间。
  - 任务回到当前用户待办列表。
  - 记录撤销完成生命周期事件。

### 5.5 用户管理

- 不做账号鉴权。
- 用户名仅作为本地数据归属标识。
- 默认用户由前端本地状态维护。
- 支持新增用户。
- 支持切换用户。
- 支持修改用户名。
- 支持删除用户。
- 删除用户时必须弹出确认提示，明确告知：
  - 删除后该用户数据不可恢复。
  - 删除会移除该用户下的任务和生命周期记录。
- 删除当前用户后，应自动切换到剩余用户；若无用户则回到默认用户。

### 5.6 菜单

- 菜单按钮使用三个竖点样式。
- 菜单项包括：
  - 历史任务。
  - 用户管理。
  - 便签大小。
  - 便签主题。
  - 便签位置。
  - 开机自启动。
  - 产品简介。
  - 退出。
- 便签大小、主题、位置使用下拉选择，减少菜单展开空间。

### 5.7 主题

- 默认主题：黄色。
- 可选主题：
  - 黄色。
  - 蓝色。
  - 绿色。
  - 粉红。
  - 白色。
- 主题选择应即时生效。
- 主题配置保存在本地浏览器存储中。

### 5.8 便签大小

- 支持小、中、大三档。
- 切换大小后调整窗口尺寸。
- 大小配置保存在本地浏览器存储中。

### 5.9 便签位置

- 支持位置：
  - 右上。
  - 左上。
  - 右下。
  - 左下。
  - 居中。
- 切换位置后调整窗口位置。
- 位置配置保存在本地浏览器存储中。

### 5.10 开机自启动

- 在菜单中提供开机自启动开关。
- 开关状态应读取当前系统配置。
- Windows 下通过当前用户注册表实现：
  - 路径：`HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
  - 键名：`StickyNote`
- 自启动状态读取不得阻塞任务列表首屏加载。
- 启动流程要求优先加载任务数据，再异步读取窗口布局和自启动状态。

### 5.11 产品简介

- 菜单入口名称：产品简介。
- 页面不展示额外页面标题。
- 展示字段：
  - 产品名称：桌面便签（本地开源版）/ StickyNote。
  - 当前版本：`v1.0 - YYYY-MM-DD HH:mm`。
  - 收费方式：本地版免费开源。
  - 开发者：miczhang（个人开发者）。
  - 数据说明：便签数据默认仅保存在本机，不上传至服务器。
  - 服务规划：云同步服务将作为独立的可选订阅服务推出。
  - 源码仓库：https://github.com/miczhang007/DSN。
  - 隐私政策：链接至仓库中的 `PRIVACY.md`。
  - 联系方式：miczhang007@qq.com。

### 5.12 退出

- 菜单中提供退出入口。
- 点击后关闭应用进程。

## 6. 数据需求

### 6.1 任务数据

任务字段：

- `id`：任务唯一标识。
- `owner`：用户归属。
- `title`：任务内容。
- `deadline_at`：截止时间，可为空。
- `is_urgent`：是否紧急。
- `created_at`：创建时间。
- `updated_at`：更新时间。
- `completed_at`：完成时间，可为空。
- `archived_at`：归档时间，可为空。

### 6.2 生命周期数据

生命周期字段：

- `id`：事件唯一标识。
- `task_id`：关联任务 ID。
- `event_type`：事件类型。
- `before_value`：变更前值，可为空。
- `after_value`：变更后值，可为空。
- `created_at`：事件创建时间。

### 6.3 用户数据

- 当前没有独立用户表。
- 用户列表由前端本地存储维护。
- 后端任务数据通过 `owner` 字段区分用户。
- 删除用户时，后端按 `owner` 删除任务和对应生命周期事件。

## 7. 界面与交互要求

- 顶部栏保持贴在顶部。
- 首页左上角展示产品名称：桌面便签。
- 产品名称同时保留在右下角品牌位置。
- 用户管理页和产品简介页不展示页面标题。
- 可点击控件不切换鼠标手型，保持默认鼠标样式。
- 右上角添加按钮和菜单按钮高度保持一致。
- 页面标题不要过大，不应冲击待办主体内容。
- 整体界面保持简洁，避免重复内容。

## 8. 非功能需求

### 8.1 性能

- 应用启动后应优先加载当前用户任务列表。
- 自启动状态读取、窗口设置等非核心操作应延后执行。
- 本地数据操作应保持轻量，避免明显卡顿。

### 8.2 本地化

- 应用界面默认使用中文。
- 产品英文名仅在产品简介等品牌信息中展示。

### 8.3 数据安全

- 所有任务数据保存在本机 SQLite 数据库。
- 不提供云同步。
- 不提供远程备份。
- 删除用户数据不可恢复，必须在交互层明确提示。

### 8.4 安装与图标

- EXE 和安装包使用黄色便签加清单图标。
- 安装后桌面图标应与应用图标保持一致。
- 安装包和 EXE 文件名统一使用 `stickynote`。

## 9. 构建与发布要求

### 9.1 构建前检查

- 检查 `src/App.vue` 中的 `versionLabel` 是否已同步到最新应用代码修改时间。
- 检查 `src-tauri/tauri.conf.json` 中 `beforeBuildCommand` 是否恢复为 `npm run build`。
- 检查是否存在正在运行的 `outputs/stickynote.exe`，如存在需先关闭，否则会影响产物覆盖。

### 9.2 推荐构建流程

1. 执行前端构建：`npm run build -- --debug`。
2. 临时将 Tauri 配置中的 `beforeBuildCommand` 置空，避免重复构建触发路径问题。
3. 执行 Tauri 打包：`npm run tauri:build`。
4. 恢复 `beforeBuildCommand` 为 `npm run build`。
5. 复制产物到 `outputs`：
   - `src-tauri/target/release/stickynote.exe` -> `outputs/stickynote.exe`
   - `src-tauri/target/release/bundle/nsis/Desktop Sticky Note_1.0.0_x64-setup.exe` -> `outputs/stickynote-setup.exe`

### 9.3 发布产物

- 单文件可执行程序：`outputs/stickynote.exe`。
- Windows 安装包：`outputs/stickynote-setup.exe`。
